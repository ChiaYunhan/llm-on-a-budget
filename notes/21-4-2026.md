## What I Learned

> (It was actually 3 days ago.)

These aren't necessarily new to me in LLM world, but they are for local hosting.

### 1. Context Windows

With cloud models, the default context windows are already set to 200K or 1M and it wasn't something I had to explicitly set in Claude Code or other AI CLIs/IDEs, so I never had to think about configuring it. That's why my subsequent messages didn't have context from previous messages.

I was asking it to show me how to print `"Hello World"` in Python, and then in the next message asking it to write to a Python file, and it would respond with:

> "What would you like to write in the Python file?"

Then I figured out it was my configuration issue with Ollama, not the model itself, when I saw `truncating input prompt` in the logs. That was surprising because my messages weren't that long, so I didn't initially think that was the problem.

#### How I fixed it

I created new model variants with larger context windows using Ollama's interactive `/save` workflow — no Modelfile needed:

```bash
ollama run gemma4:e4b
```

Then inside the REPL:

```
>>> /set parameter num_ctx 32768
>>> /save gemma4:e4b-32k
>>> /bye
```

Same idea for Qwopus with `num_ctx 65536` saved as `Qwopus3.5:9b-65k`.

Under the hood, `/save` auto-generates a Modelfile and runs `ollama create` on it. The new models point to the **same underlying blobs** as the originals, so the disk sizes match exactly (6.5 GB / 9.6 GB) — nothing gets duplicated. You can inspect the generated Modelfile with:

```bash
ollama show gemma4:e4b-32k --modelfile
```

One caveat: the `num_ctx` set here is just the **default** — clients can still override it per-request. And bumping the context window increases KV cache memory at runtime, which ties right back to the KV Cache note below.

### 2. Quantization

Quantization isn't new to me, but I definitely need to learn more about it.

- Lower bit-width → faster inference, but less precision
- The weights are essentially less precise — it's like rounding `0.184284` to `0.2`
- You're losing a lot of information about that number, but now you don't have to consider many decimal points and it's easier to add to another number

That's how I think about it: the lower the quantization, the closer horses and donkeys are to each other — they're both 4-legged and can pull a wagon.
